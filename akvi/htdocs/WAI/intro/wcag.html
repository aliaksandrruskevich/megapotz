Указания по обеспечению доступности Веб-контента (WCAG)

Документы с указаниями по обеспечению доступности Веб-контента (WCAG) объясняют, как сделать Веб-контент доступным для людей с ограниченными возможностями. Под словом "Веб-контент" подразумевается информация на Веб-странице или в Веб-приложении, включая текст, картинки, формы отправки, звуки и прочее.

WCAG является частью серии указаний по доступности, включающих Указания по обеспечению доступности инструментов авторинга (ATAG), и указания по доступности браузеров (UAAG). Основные компоненты Веб-доступности объясняет, как различные указания связаны между собой.

Для кого предназначен WCAG

WCAG в первую очередь предназначен для:

  • Веб-разработчиков (верстальщиков, дизайнеров, и т.д.)
  • Разработчиков инструментов Веб-авторинга
  • Разработчиков инструментов оценки доступности Веб-страниц
  • Других людей, которым нужен или которые хотят иметь стандарт для Веб-доступности

WCAG и связанные ресурсы предназначены для удовлетворения потребностей различных аудиторий, включая тех, кто только начал заниматься проблемами Веб-доступности, менеджеров, высших должностных лиц и других.

Версии WCAG: 1.0 и 2.0

Указания по обеспечению доступности Веб-контента 1.0 были опубликованы в мае 1999. WCAG 2.0 был опубликован 11 декабря 2008. WCAG 2.0 широко применим к более передовым технологиям; его проще понимать и использовать; можно более точно проводить формальное тестирование, как с помощью автоматизированных средств, так и методом человеческой оценки.

W3C WAI рекомендует использовать WCAG 2.0 вместо WCAG 1.0.

Большинству Веб-сайтов, которые соответствуют WCAG 1.0, не потребуется значительных изменений для приведения в соответствие WCAG 2.0, а некоторые, возможно, не потребуют изменений вообще. Чтобы помочь вам перейти на WCAG 2.0, WAI разрабатывает:

  • Чем WCAG 2.0 отличается от WCAG 1.0
  • Сравнение ключевых характеристик WCAG 1.0 и WCAG 2.0
  • Как обновить сайт от WCAG 1.0 до WCAG 2.0
  • Руководство по переводу политик Веб-доступности к WCAG 2.0

Что входит в WCAG 2.0

Во WCAG 2.0 входят 12 указаний, которые организованы по четырём принципам: воспринимаемость, управляемость, понятность и надежность. Для каждого указания есть проверяемые критерии успеха, которые состоят из трёх уровней: A, AA, и AAA.

С кратким содержанием указаний WCAG 2.0 можно ознакомиться в статье Взгляд на WCAG 2.0.

Большинство людей будут использовать вспомогательные материалы при разработке Веб-контента и Веб-инструментов, вместо технической документации стандарта. Вспомогательные материалы по WCAG 2.0 включают:

  • Как соответствовать WCAG 2.0: Быстрым руководством по требованиям (критериям успеха) и техникам указаний по доступности Веб-контента 2.0 является список проверки WCAG 2.0
  • Понимание WCAG 2.0 содержит дополнительные указания по обучению и применению WCAG 2.0 для тех, кто хочет понять принципы и критерии успеха более тщательно.
  • Техники WCAG 2.0 дает конкретную информацию о том, как создавать доступный Веб-контент, такую как примеры HTML-кода.

Для более подробной информации о том, какое отношение эти документы имеют друг к другу и как они взаимосвязаны, см. Документы WCAG 2.0.

Самое лучшее место для начала работы с WCAG 2.0 Как соответствовать WCAG 2.0 (Краткий справочник).

Дополнительные материалы

Обучающие материалы, относящиеся к WCAG 2.0, перечисленные в области навигации этой страницы.

WAI планирует публикацию дополнительных материалов, которые помогут Веб-разработчикам, не являющимся экспертами в области Веб-доступности, разрабатывать доступное Веб-содержимое, которое соответствует WCAG 2.0. Возможно, будет написана статья "Заметки по применению" (рабочее название) с рекомендациями по конкретным темам, таким как изображения, ссылки или таблицы. Например, "Заметка по применению" о формах отправки начиналась бы с простых примеров и включала бы критерии успеха WCAG 2.0, приемы и стратегии для создания доступных форм отправки.

WAI планирует периодически обновлять документы из серии Понимание WCAG 2.0 и Техники WCAG 2.0. Мы приветствуем комментарии и отправку нам новых техник.

Технический формат документа

WCAG следует формату, принятому W3C для технических спецификаций, который включает в себя несколько разделов в начале: ссылки на различные версии, список редакторов, авторские права, аннотацию и статус документа со ссылкой на список опечаток, а также адрес электронной почты для комментариев. Большинство спецификаций WAI имеют ссылку на содержание в верхней части страницы.

Кто разрабатывает WCAG

Технические документы WCAG разрабатываются Рабочей группой по доступности Веб-контента (РГ WCAG), которая входит в отдел Инициативы по Веб-доступности (WAI) Консорциума World Wide Web (W3C). Для получения дополнительной информации о деятельности Рабочей группы, см. страницу РГ WCAG.

В документе Как WAI разрабатывает указания по Веб-доступности через процесс W3C описаны вехи процесса разработки и формальные периоды для общественного рассмотрения. Возможности для рассмотрения и представления комментариев по документам WAI, находящимся в разработке, объявляются на домашней странице WAI и через список рассылки WAI Interest Group. Существует также форма для отправки комментариев по существующим документам WCAG 2.0 и форма для отправки новых техник.

Краткий обзор возможностей для содействия WCAG и другим работам WAI содержится в документе Участие в WAI.
